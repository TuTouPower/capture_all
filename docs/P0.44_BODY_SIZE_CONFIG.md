# P0.44 Body 大小限制可配置

> 状态：未修复
> 日期：2026-06-13

## 现状

请求体限制 **10KB**、响应体限制 **50KB**，硬编码在 `src/shared/constants.ts`：

```typescript
export const MAX_REQUEST_BODY_BYTES = 10 * 1024;   // 10KB
export const MAX_RESPONSE_BODY_BYTES = 50 * 1024;  // 50KB
```

上次采集 202 条请求中 16 条因超 50KB 被截断（均为 JS/CSS bundle），最大实际 body 均卡在 50KB 上限。

## 改动

### 1. 默认值改为 1MB

`src/shared/constants.ts`:
- `MAX_REQUEST_BODY_BYTES` → `1024 * 1024`（1MB）
- `MAX_RESPONSE_BODY_BYTES` → `1024 * 1024`（1MB）

### 2. UserConfig 新增字段

`src/shared/types.ts` — `UserConfig` 新增：
```typescript
max_request_body_bytes: number;
max_response_body_bytes: number;
```

`src/shared/constants.ts` — `DEFAULT_USER_CONFIG` 新增：
```typescript
max_request_body_bytes: 1048576,  // 1MB
max_response_body_bytes: 1048576, // 1MB
```

### 3. 采集代码改为从 config 读取

`src/background/network_capture.ts` — 4 处引用改为从 config 对象读取限制值（不再用全局常量）。

`NetworkCaptureConfig` 接口新增对应字段，`start_network_capture` 传入用户配置的限制。

### 4. 设置 UI

`src/dashboard/dashboard.ts` — 在「采集默认值」section 追加两个数字输入框：

```html
<div class="field"><span class="field-lbl">请求体上限 (字节)</span>
    <input class="input mono" type="number" data-cfg="max_request_body_bytes"
     value="1048576" min="1024" max="104857600" step="1024"></div>
<div class="field"><span class="field-lbl">响应体上限 (字节)</span>
    <input class="input mono" type="number" data-cfg="max_response_body_bytes"
     value="1048576" min="1024" max="104857600" step="1024"></div>
```

`wire_settings()` 中已有 `log_max_entries` 的 `Number(v)` 转换模式，参照添加。

### 5. 测试

验证新字段的 save/load，以及采集时使用配置的限制值。

## 影响文件

| `src/background/network_capture.ts` | 4 处从常量改为从 config 读取 |
