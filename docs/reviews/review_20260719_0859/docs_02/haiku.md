## 高优先级

### P1-01：缺少"决策"占位节

  - **位置**：`spec.md` 第 3 行（"## 背景"）与第 7 行（"## 范围"）之间
  - **现象**：模板只包含 背景、范围、非范围、验收标准、依赖与约束 五个节。唯一活跃 spec（`T008_label_routing/spec.md`）在"背景"与"范围"之间插入了"## 决策（用户确认）"节，用于记录用户确认的非显然决策。`CLAUDE.md` 第 5 行明确要求"非显然决策追加到 `docs/blueprint/decisions.md`"，但 spec 中先行记录决策是合理的实践——plan 阶段提出的决策在 spec 中展示，经用户确认后再在 finalization 阶段落入 `decisions.md`。
  - **影响**：模板与实际使用不一致；新 task 可能遗漏决策记录，导致关键取舍信息散落在对话上下文中，后续接手者无法回溯。
  - **建议**：在"## 背景"与"## 范围"之间增加：

    ```
    ## 决策（用户确认）

    - {用户已确认的非显然取舍；无则写"无"。}
    ```
  - **置信度**：高（有确切实例佐证）
  - **级别**：高

### P1-02：验收标准缺少可验证性指导

  - **位置**：`spec.md` 第 17 行
  - **现象**：模板验收标准仅写 `- [ ] {可独立验证的完成条件。}`，未提示每条验收标准应满足什么质量标准。`CLAUDE.md` 第 4 行要求"定义成功标准，循环直到验证通过"，但模板未引导作者写出可测量、可验证的验收条件。例如 T008 的验收标准包含可执行命令（`grep -rn`、`npx tsc --noEmit`、`npm test`），但模板未提示这一维度。
  - **影响**：新手或 rush 状态下可能写出模糊验收标准（如"功能正常"、"无 bug"），导致 review 环节无法有效判断是否达标。
  - **建议**：在 `- [ ]` 行下方增加一行注释提示：

    ```
    - [ ] {可独立验证的完成条件。优先用可执行命令或可测量指标（如 grep、test pass、build 成功、UI 可观察行为）。}
    ```
  - **置信度**：中（符合 TDD 精神，但模板是骨架，不一定需要过度说明）
  - **级别**：高

## 中低优先级

### P2-01：模板标题不含任务编号占位符

  - **位置**：`spec.md` 第 1 行
  - **现象**：模板标题为 `# Task spec`；实际使用为 `# Task spec — T008 label_routing`。模板未给出编号占位符提示。
  - **影响**：微小——作者会自然加编号，但增加占位符可降低遗忘概率。
  - **建议**：改为 `# Task spec — {TNNN slug}` 或至少在大括号注释中提示。
  - **置信度**：中
  - **级别**：低

### P2-02：缺少与 plan/log 模板的交叉引用

  - **位置**：`spec.md` 末尾
  - **现象**：spec、plan、log 三模板构成 task 文档三元组，但 spec 模板未提及另外两个模板的存在或用途。`CLAUDE.md` 第 4 行写"spec 和 plan 先行，一起写完交用户一次性审核"，模板可提醒此流程。
  - **影响**：微小——`CLAUDE.md` 已有流程描述，但模板内提示可降低查阅负担。
  - **建议**：末尾加一行注释：

    ```
    <!-- 与 plan.md、log.md 配套使用；spec + plan 写完一并提交审核。 -->
    ```
  - **置信度**：低（注释对 AI agent 可读但对人可能无用）
  - **级别**：低

### P2-03：依赖与约束节未区分"即时依赖"与"长期约束"

  - **位置**：`spec.md` 第 19-21 行
  - **现象**：模板将"前置依赖"与"平台/安全/兼容性约束"合并为一个项目。T008 实际使用中在前一条写了已完成的前置 task，在后一条写了不变量（Bridge 仅绑 127.0.0.1 等）。这两个概念性质不同：前置依赖是临时的（task 完成后解除），约束是长期的（贯穿项目始终）。
  - **影响**：微小——当前合并写法可接受，但分列可提升清晰度。
  - **建议**：拆为两条：

    ```
    - 前置依赖：{需先完成的 task 或外部条件；无则写"无"。}
    - 约束：{平台、安全、兼容性等长期限制；无则写"无"。}
    ```
  - **置信度**：低（属于模板优化而非缺陷）
  - **级别**：低
