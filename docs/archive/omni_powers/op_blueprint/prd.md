# 产品需求

## 1. 产品定位

Capture All（中文：全采）是浏览器调试黑匣子。把一次浏览器采集（capture）中的用户操作、页面导航、网络请求、控制台日志、运行时错误、Storage 变化、Cookie 变化等结构化证据，记录成可回看、可分析、可导出、可被本地 AI Agent 查询的调试证据包。

定位：浏览器采集证据记录器，本地优先的浏览器问题取证与分析工具。

不是录屏工具，不是 Chrome DevTools 替代品，不是弱版 Dashboard。

核心价值：把问题发生前后的所有浏览器证据按时间和因果关系组织，让人和 AI 都能快速理解发生了什么。

## 2. 目标用户

| 用户类型 | 核心需求 |
|---|---|
| 开发者 | 问题复现时快速获取结构化证据、导出、AI 辅助分析 |
| 测试 / QA | 一键记录复现过程，导出可复核材料 |
| 技术支持 | 收集用户侧问题证据，减少来回确认 |
| AI Agent 用户 | 让本地 Agent 查询历史采集，获取完整上下文 |

## 3. 信息架构

只有两个用户可见界面：

- **弹出窗口（Popup）**：轻量采集控制面板。开始/停止采集、当前采集状态、7 数据标签实时统计、最近采集列表快捷入口。不是配置页、不是数据分析页、不是 Dashboard。
- **主面板（Dashboard）**：完整采集工作台。采集历史列表、概览统计、单次采集详情（时间线、网络、控制台、证据等）、设置、导出。

Agent / MCP 是本地桥接和工具接口，供 AI Agent 查询状态、读取数据和导出，不是用户界面。

## 4. 核心功能

### 4.1 弹出窗口三状态

- 状态 1 开始采集：整宽蓝色大按钮，下方 7 标签仅图标+名称（无数字）。
- 状态 2 采集中：红色计时按钮（整块即结束键）+ 实时详情按钮；7 标签显示实时计数。
- 状态 3 采集完成：绿色时长块 + 查看详情/开始新采集；7 标签显示计数。

操作区固定高度 108px，三种状态按钮行高一致。宽度约 360-400px，不出现垂直滚动条。

### 4.2 7 数据标签

UI 层统一 7 个面向用户的标签，弹出窗口和主面板口径完全一致，计数来自实际数据：

| # | 标签 | 覆盖内容 |
|---|---|---|
| 1 | 用户行为 | 鼠标、键盘、滚动、输入、剪贴板、焦点、表单提交、全屏、打印、resize、visibility |
| 2 | 页面导航 | 页面加载、URL 变化、route 变化、Tab 创建/切换、dom_ready、visibility_change |
| 3 | 网络请求 | Fetch/XHR/WebSocket、请求体、响应体状态 |
| 4 | 控制台 | console.log/warn/error/debug |
| 5 | 错误异常 | 运行时异常、未处理 Promise、资源错误、网络失败、采集错误 |
| 6 | Storage | localStorage / sessionStorage 变化 |
| 7 | Cookie | Cookie 新增、修改、删除、过期 |

脱敏是配置项，不是数据标签。

### 4.3 主面板

- 左侧边栏：采集记录 / 当前采集 / 导出任务 / 设置。
- 采集列表页：概览统计卡（总量、事件量、异常/风险），不展示"当前采集中"卡片，不展示历史采集分层统计；列表表格不展示"模式"列、不提供模式筛选；批量导出/删除。
- 采集详情页：在主面板内打开，不跳转独立页面。面包屑 + 7 标签统计 + Tab 切换（概览/时间线/网络/控制台/证据/存储/Cookie/配置）+ 三栏布局（筛选 / 事件列表 / 检视器）。

### 4.4 采集核心

- 同一时间只允许一次活跃采集。
- 开始：创建 CaptureRecord，通知 content script 激活，按需 attach CDP（console / exception / body），启动 agent bridge 轮询。
- 停止：停止所有采集模块，detach CDP，flush 所有未写入数据，更新 status=completed。
- 实时计数：采集中 7 标签数字随采集实时变化，来自实际数据。

### 4.5 导出

| 格式 | 说明 |
|---|---|
| JSON | 完整结构化数据（capture_id + events 全量） |
| JSONL | 逐行事件，适合 AI ingest |
| HAR | 网络请求归档，兼容 Chrome DevTools / Fiddler / Charles |
| HTML | 自包含可读报告，必须转义防 XSS |

### 4.6 设置

通用（主题、语言、时区）、隐私与脱敏、导出（文件名模板、保存位置）、存储管理。

### 4.7 AI Agent / MCP

- Bridge 仅绑定 127.0.0.1，token 用户提供。
- Agent 能力：查询状态、开始/停止采集、查询采集列表和详情、获取时间线、按分类查询数据、导出。
- 工具层默认不替用户删除、过滤或摘要数据。

详见 `specs/agent_mcp.md`。

## 5. 成功标准

- 弹出窗口三种状态切换正确，无滚动条，宽度合适。
- 7 标签在弹出窗口和主面板名称与计数完全一致。
- 采集开始/停止真实生效，采集中计数实时更新（非全 0）。
- 主面板采集详情在主面板内打开，时间线/网络/控制台 Tab 有实际数据。
- 导出 JSON/JSONL/HAR/HTML 正确，HTML 无 XSS。
- Agent 可通过 MCP 查询状态、采集列表、详情、时间线、分类数据，触发导出。
- UI 不出现"深度采集/标准采集/模式切换/当前采集中"等旧概念。

## 6. 明确不做

- 云端同步、团队协作后台、公网 Agent 服务。
- 视频录屏、浏览器状态 Replay（回放）。
- 完整 DOM diff 工具。
- 自动问题诊断替代人工判断。
- 深度采集 / 标准采集 模式切换。
- compact 密度选项（仅 regular）。
- "当前采集中"统计卡片。
- 模式筛选、模式列、模式徽章。
- `detail.html` 独立详情页（已合并入 dashboard）。
