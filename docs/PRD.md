# Capture All 全采 -- 产品需求文档

版本：v1.0
产品阶段：Chrome MV3 扩展工程原型 -> 专业调试工具化
文档日期：2026-06-09

---

## 1. 产品概述

### 1.1 产品名称

- 英文：Capture All
- 中文：全采

### 1.2 产品一句话

全采是浏览器调试黑匣子，把一次浏览器 Session 中的用户操作、页面导航、网络请求、控制台日志、运行时错误、Storage 变化、Cookie 变化等结构化证据，记录成可回看、可分析、可导出、可被本地 AI Agent 查询的调试证据包。

### 1.3 产品定位

全采不是录屏工具，也不是 Chrome DevTools 的替代品。

定位：浏览器 Session 证据记录器，本地优先的浏览器问题取证与分析工具。

核心价值：把问题发生前后的所有浏览器证据，按照时间和因果关系组织起来，让人和 AI 都能快速理解发生了什么。

### 1.4 产品性格

- 专业调试工具风，不像普通 SaaS Dashboard
- 不像弱版 DevTools
- 不像录屏软件
- 本地优先、AI-ready

## 2. 目标用户

| 用户类型      | 核心需求                                             |
| ------------- | ---------------------------------------------------- |
| 开发者        | 问题复现时快速获取结构化证据、导出、AI 辅助分析      |
| 测试/QA       | 一键记录复现过程，导出可复核材料                     |
| 技术支持      | 收集用户侧问题证据，减少来回确认                     |
| AI Agent 用户 | 让本地 Agent 查询历史 Session，获取完整上下文         |

## 3. 信息架构

全采只有两个用户可见界面：

### 3.1 弹出窗口（Popup）

轻量采集控制面板。负责：
- 开始/停止采集
- 展示当前采集状态
- 展示 7 个数据标签的实时统计
- 最近采集列表快捷入口

弹出窗口不是配置页、数据分析页、Dashboard。

### 3.2 主面板（Dashboard）

完整采集工作台。负责：
- 采集历史列表
- 概览统计
- 单次采集详情（时间线、网络、控制台、证据等）
- 设置
- 导出

Agent / MCP 是本地桥接和工具接口，供 AI Agent 查询状态、读取数据和导出，不是用户界面。

## 4. 核心使用场景

### 场景 1：开发者复现线上问题

1. 打开弹出窗口，点击「开始采集」
2. 在页面上操作复现问题
3. 弹出窗口实时显示各数据标签计数
4. 点击红色计时按钮停止采集
5. 点击「查看详情」进入主面板
6. 在时间线中定位失败请求和附近事件
7. 导出 JSON/HAR 发给同事

### 场景 2：测试人员记录测试证据

1. 打开弹出窗口，点击「开始采集」
2. 按测试用例操作页面
3. 停止采集
4. 点击「查看详情」查看证据链
5. 导出 HTML 报告作为测试证据归档

### 场景 3：AI Agent 查询历史采集

1. 用户已完成若干次采集
2. 本地 Agent 通过 MCP 查询采集列表
3. Agent 获取指定采集的时间线和失败请求
4. Agent 分析问题原因并给出建议

## 6. 核心功能需求

### 6.1 弹出窗口

弹出窗口是采集控制台，面积适合普通 Chrome 扩展弹窗，不应出现垂直滚动条。

#### 6.1.1 布局结构（从上到下）

1. 标题栏：左侧「Capture All 全采」，右上角「主面板」入口（open_in_new 图标）
2. 状态区 + 操作按钮区（固定高度 108px）
3. 数据标签区：7 个标签卡片，三列网格
4. 最近采集：最近 3 条，每条含时间、时长、事件数、「查看详情」

#### 6.1.2 状态一：开始采集

- 整宽蓝色大按钮「开始采集」（52px 高，14px 圆角，#3b82f6 主色）
- 下方 7 个数据标签卡片，仅显示图标+名称，无数字、无分隔线
- 三列网格：第一行 3 个、第二行 3 个、第三行 1 个（Cookie），无占位卡

#### 6.1.3 状态二：采集中

- 红色计时按钮，显示格式 `00:03:28`，含「点击结束」提示，整块即结束键
- 右侧白色描边按钮「实时详情」
- 下方 7 个标签显示数字统计：图标 + 名称 + 分隔线 + 数字
- 计数来自实际采集数据，持续更新
- 三个状态格子大小统一

#### 6.1.4 状态三：采集完成

- 绿色时长块（含勾选标记）
- 右侧两按钮叠放：「查看详情」「开始新采集」
- 下方 7 个标签显示数字统计（与采集中一致）

#### 6.1.5 视觉约束

- 右上角主面板按钮背景色与外部一致（白色）
- 「查看全部」按钮与「查看详情」按钮视觉对齐
- 最近采集列表不展示历史采集分层徽章
- 不出现"深度采集"/"标准采集"/"就绪"字样

### 6.2 主面板

#### 6.2.1 导航

左侧 sidebar：采集记录 | 导出任务 | 设置 | MCP 集成

#### 6.2.2 采集记录页

- 概览统计卡（聚焦采集总量、事件量、异常/风险），不展示"当前采集中"卡片
- 不展示历史采集分层统计卡
- 采集列表（可排序表格），不展示"模式"列、无模式筛选
- 批量操作（导出、删除）

#### 6.2.3 采集详情页

点击采集条目在主面板内打开详情，不跳转到独立页面。

- 面包屑导航返回采集记录
- 7 个数据标签统计，与弹出窗口统一口径
- 标签：用户行为、页面导航、网络请求、控制台、错误异常、Storage、Cookie

**Tabs：**

| Tab    | 中文   | 内容                                      |
| ------ | ------ | ------------------------------------------ |
| 概览   | 概览   | 本次采集摘要、关键指标                     |
| 时间线 | 时间线 | 所有事件按时间排序，核心视图               |
| 网络   | 网络   | 请求列表、请求详情                         |
| 控制台 | 控制台 | 日志和错误                                 |
| 证据   | 证据   | 用户行为、导航、Storage、Cookie            |
| 存储   | 存储   | localStorage/sessionStorage 变化           |
| Cookie | Cookie | Cookie 变化                                |
| 配置   | 本次配置 | 本次采集的配置快照                       |

**时间线要求：**
- 融合多数据源（用户操作、网络、控制台、错误、Storage、Cookie）
- 每条事件包含相对时间和绝对时间
- 失败请求和 Console error 明显突出（红色）
- 点击事件展开右侧 Inspector 展示详情和附近相关事件

### 6.3 数据采集标签

**7 个标签（最终版本）**：

| 序号 | 中文标签 | 说明                                     | 颜色   |
| ---- | -------- | ---------------------------------------- | ------ |
| 1    | 用户行为 | 鼠标、键盘、滚动、输入等页面交互         | 蓝色   |
| 2    | 页面导航 | 页面加载、URL 变化、Tab 创建/切换等      | 蓝紫色 |
| 3    | 网络请求 | Fetch、XHR、请求体、响应体状态           | 紫色   |
| 4    | 控制台   | console.log/warn/error/debug 等         | 橙色   |
| 5    | 错误异常 | 运行时异常、未处理 Promise、资源错误     | 红色   |
| 6    | Storage  | localStorage、sessionStorage 变化       | 绿色   |
| 7    | Cookie   | Cookie 新增、修改、删除、过期            | 青蓝色 |

**标签组件规范：**
- 每个卡片圆角 14px，内边距 16px，间距 12px
- 浅色背景 + 对应颜色边框
- 图标在左，名称其后
- 数字靠右，字号略大，字重 600
- 三种状态格子大小统一

### 6.4 导出

| 格式 | 说明                             |
| ---- | -------------------------------- |
| JSON | 完整结构化数据                   |
| JSONL| 逐行事件，适合 AI ingest         |
| HAR  | 网络请求归档，兼容 DevTools      |
| HTML | 自包含可读报告（必须转义防 XSS） |

### 6.5 设置

- 通用：主题、语言、时区
- 隐私与脱敏
- 导出：文件名模板、保存位置
- 存储管理

已废弃：默认模式（标准/深度）选择项。

### 6.6 AI Agent / MCP

- Bridge 仅绑定 127.0.0.1
- Token 认证
- Agent 能力：查询状态、开始/停止采集、查询采集列表和详情、获取时间线、按分类查询数据、导出
- 工具层默认不替用户删除、过滤或摘要数据

## 7. 非功能需求

### 7.1 技术约束

- Chrome MV3 扩展
- 原生 HTML / CSS / TypeScript（非 React）
- 构建：Vite + @crxjs/vite-plugin
- 存储：IndexedDB（采集数据）+ chrome.storage.local（用户配置）
- 遵循 Chrome Extension CSP
- TypeScript strict，命名 snake_case，缩进 4 空格
- 构建输出 `artifacts/dist/`

### 7.2 性能

- 大数据量页面避免一次性渲染卡顿
- 时间线后续需虚拟滚动或分批渲染
- 弹出窗口无滚动条

### 7.3 视觉规范

- 浅色卡片风，白色背景，圆角大卡片，轻微阴影
- 主色：蓝色 #3b82f6
- 字体：IBM Plex Sans（正文）+ IBM Plex Mono（数字、URL、代码）
- 密度：仅 regular

### 7.4 隐私与安全

- 默认开启脱敏
- 敏感字段必须脱敏：Authorization、Cookie、Set-Cookie、Token、API Key、Password
- 请求体、响应体、控制台输出按配置处理
- HTML 导出必须转义动态内容
- 不硬编码 secret、token、弱口令

### 7.5 国际化

- 支持中文、英文
- 使用 data-i18n 属性系统

### 7.6 主题

- 浅色、深色、跟随系统
- 浅色为默认

## 8. 用户故事

**US-01：快速开始采集**
作为开发者，打开弹出窗口后一眼就能看到「开始采集」按钮，点击即开始，不需要先选择模式或配置参数。
验收：弹出窗口默认显示蓝色「开始采集」按钮，点击后立即进入采集中状态，计时器开始计时。

**US-02：实时监控采集数据**
作为测试人员，在采集中能实时看到各类数据的计数，了解目前采集了多少内容。
验收：采集中 7 个标签显示实时数字，数字来自实际数据并随采集更新。

**US-03：停止采集并查看详情**
作为开发者，停止采集后能快速进入详情页查看时间线和所有采集证据。
验收：采集完成后显示「查看详情」按钮，点击进入主面板详情页，时间线为默认视图。

**US-04：时间线分析问题**
作为开发者，在统一时间线中定位第一个失败请求，查看附近的相关事件。
验收：时间线按相对时间排序所有事件，失败/错误以红色突出，点击事件展开 Inspector 展示详情和附近事件。

**US-05：导出证据**
作为 QA，采集完成后能导出结构化数据发给开发者或存档。
验收：主面板详情页提供导出按钮，支持 JSON/JSONL/HAR/HTML，导出前如有敏感数据给出提示。

**US-06：最近采集入口**
作为用户，在弹出窗口底部看到最近几次采集，快速进入之前的详情。
验收：弹出窗口底部展示最近 3 条采集记录，点击「查看详情」进入主面板对应详情页。

**US-07：AI Agent 查询**
作为 AI Agent 用户，本地 Agent 能通过 MCP 接口查询历史采集数据。
验收：Agent 可查询采集列表、详情、时间线、分类数据，可触发导出。

## 9. 优先级排序

### P0：当前核心（本轮对齐）

- 弹出窗口三种状态（开始采集、采集中、采集完成）
- 7 数据标签统一展示（弹出窗口和主面板口径一致）
- 弹出窗口无模式切换（去除所有深度/标准采集 UI）
- 主面板采集记录（去除模式列、模式筛选、分层统计卡）
- 主面板采集详情（7 标签统计、时间线、网络、控制台等）
- 采集开始/停止真实逻辑
- 实时计数更新（来自实际数据）
- 整体视觉统一（主色 #3b82f6，regular 密度）

### P1：后续增强

- 证据视图完善（用户行为、导航、Storage、Cookie 详情）
- 导出流程完善（敏感数据提示、格式说明）
- HTML Report 优化（防 XSS、自包含可读报告）
- MCP Bridge 状态 UI
- 设置页面完善

### P2：AI-ready 与高级能力

- Agent 可读数据说明与风险提示
- 时间线虚拟滚动（大 Session 性能优化）
- 多 Session 对比分析
